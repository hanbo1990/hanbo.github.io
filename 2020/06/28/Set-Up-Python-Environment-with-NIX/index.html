Set Up Python Environment with NIX

Posted on 2020-06-28 Edited on 2020-10-16 In other

In the example, we are going to create a nix-shell environment for python crawler.

Following dependencies are used:

  • python 3.7
  • pylint, for checking code format
  • python package beautifulsoup4 and requests

Prepare your nix-shell environment

Search your dependencies in nix store

The first thing we need to do is go to Search Nix Packages to search these packages. This website contains all available packages in the nix channel which your local nix-store subscribes to. All packages you find in the channel can be directly downloaded to your store. Packages you cannot find need some extra work before they can be added to the nix-store, a demo will be shown later when we create environment for C/C++.

Take python 3.7 for example, we can find it’s attribute name is python37.

After this step, we should get all dependencies as below.

Dependencies Attribute Name
python 3.7 python37
pylint python37Packages.pylint
beautifulsoup4 python37Packages.beautifulsoup4
requests python37Packages.requests

Create nix-shell definition

Now we have defined a shell with all dependencies we need. Note here, for python, python38Packages.requests is equal to python38.pkgs.requests.

Check your nix-shell

Before we start the shell, let’s first check what we have in the system.

So none of python3.7 and beautiful soup is in my environment. Let’s start the shell.

You can find a standalone shell is created starting with nix-shell: and in this shell, you can only find python 3.7. The required dependencies also present. One thing to notice is the nix-shell environment does not pollute the system environment or file systems.

The full example can be found here. To make things more development-friendly, I will also help you set up your vs-code.

Setup VS-Code

Install extensions

Two extensions are required here:

  1. Python as always required.
  2. Nix Environment Selector To automatically configure the environment path. In this case, the path to python executable.

Configure your linting

Normally Nix Environment Selector will configure everything for you, but for linting, you need to set the path yourself.

In the configuration for Python extension, set the path of pylint to the result you find in your nix-shell environment by which pylint. Mine is /nix/store/a4k2i7di4738c8pcmcch21vrzvkjwz75-python3.7-pylint-2.4.4/bin/pylint.

To this point, you should have everything set up for your development. You can browse the code (with jump support), execute and debug code. And all dependencies stay in and go with your development folder!
