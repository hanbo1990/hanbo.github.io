use section to manage init

Posted on 2020-10-16 In ccpp

This page shows how to use sections to do initialization.

The idea is simple, store function pointers in sections and call them by accessing the section address. This approach is commonly used in Linux (module_init as an example) and also RTOS where multiple levels of init are required.

The example below is built with gcc and running on a Linux machine. The theory behind is the same in embedded devices but the best way to implement it can be a bit different due to the linker you are using.

Problems in example

The above example is quite commonly used. We get a list of modules with the same interface, and the module manager is created to init the modules. And module manager is inited by the main function.

So every time a new module is created or removed, the module manager needs to be updated or we need to use ifdef to exclude some of the modules from the build. This dependency makes the code hard to maintain. So is it possible that the module registers itself to the module manager?

The idea is using section.

Find start and stop of your section

Once we have the pointers available, the next step is to read out and execute the pointers. Since all the functions have the same type, it’s easy to transform the function address of functions. But we still have 1 problem here:

How to get the start and end address of a section?

Normal case (use generated symbols)

In the test environment, the start and stop of the section can be accessed with __start_SECTION and __stop_SECTION where SECTION is the name we just created. The section name cannot start with . in this case.

If in the worst case, it does not work?

Use Linker script to manually define them

Linker script provides you more control of the memory layout of your sections. You can define symbols like _init_start and _init_end around your session. By doing so, you can put all stages of init together in one for loop (you will find how useful it is later). The disadvantage is you need to setup linker script for each platform which normally is fine.
